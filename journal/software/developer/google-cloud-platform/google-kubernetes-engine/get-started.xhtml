Get Started

If you are new to Kubernetes, you should probably begin by reading an overview of Google Kubernetes Engine (GKE) or its documentation. If you are ready to dive in immediately, start by following the guide to Configuring Cluster Access for kubectl.

Once you have installed and configured kubectl, you should read Deploying a Containerized Web Application. It documents some critical skills such as how to:

  • Push a Docker image to the Google Container Registry
  • Create a GKE cluster
  • Deploy a sample app to GKE
  • Expose an app to the Internet via a LoadBalancer
  • Deploy a new version of an app
